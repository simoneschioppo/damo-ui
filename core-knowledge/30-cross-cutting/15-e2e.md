# End-to-End Tests

Status: documented · Last scan: f49fab2 · Sources:
`e2e/playwright.config.ts`, `e2e/tests/scenarios/`, `e2e/package.json`.

## Summary

Repo-level Playwright suite that drives the live Next.js app at
`http://localhost:3000` in **chromium** and **webkit**. Two flavors of
test:

1. **Smoke / docs flows** — homepage, docs shell, navbar, 404 page,
   density toggle, display-settings menu.
2. **Theme-generator runtime audits** — mutate a CSS custom property
   on `:root` from the editor UI and assert the change actually
   propagates to a live consumer's computed style.

E2E owns the runtime contract that jsdom-based unit tests cannot
verify: real CSS engine, real cascade, real cross-browser
substitution semantics. See `10-library/40-testing/README.md` for
the lib's two-tier coverage model.

## Configuration

`playwright.config.ts`:

- `testDir: './tests'`
- `fullyParallel: true`
- `forbidOnly: !!process.env.CI`
- `retries: process.env.CI ? 2 : 0`
- `workers: process.env.CI ? 2 : undefined`
- `reporter: [['list'], ['html', { open: 'never' }]]`
- `use.baseURL: 'http://localhost:3000'`
- `use.trace: 'on-first-retry'`
- `use.video: 'retain-on-failure'`
- `use.screenshot: 'only-on-failure'`

Two projects: `chromium` (Desktop Chrome) and `webkit` (Desktop
Safari). Firefox is intentionally not covered — adding it has been
discussed but not prioritized.

`webServer` is **CI-aware** (PR #90):

- **Locally** (no `CI` env): boots `pnpm --filter @damo/web dev` for
  HMR and zero pre-build. Honors `reuseExistingServer: true` so a
  developer's already-running `pnpm dev` short-circuits the boot.
- **In CI** (`process.env.CI === 'true'`): boots
  `pnpm --filter @damo/web start` against a freshly-built `.next`
  (the CI workflow runs `pnpm --filter @damo/web build` immediately
  before Playwright). Production server has no HMR overhead and
  serves the first request in <1s instead of 5–10s under `next dev`.
  `reuseExistingServer: false` in CI guarantees a clean boot per run.

## Scenario tiers

### Smoke and docs flows

Cheap-to-run, broad coverage of the public-facing app.

- `smoke.spec.ts` — every primary route renders 200, no console
  errors.
- `home-hero.spec.ts`, `navbar-brand.spec.ts`,
  `not-found.spec.ts` — visual + content sanity checks.
- `docs-shell.spec.ts`, `docs-completeness.spec.ts` — every
  component documented in `/docs/components/*` actually renders the
  example sandbox.
- `density-effect.spec.ts` — flipping `[data-density]` cascades
  through Tailwind's `--spacing` rebind.
- `display-settings-menu.spec.ts` — preferences menu opens and
  applies its selections.
- `i18n-switcher.spec.ts` — docs-site language switcher (story 1.1):
  default English, switching to Italiano flips `<html lang>`,
  persists to `localStorage.locale` + `NEXT_LOCALE` cookie, and
  re-renders the docs chrome (nav, sidebar groups) in Italian after
  the switcher's reload.
- `favicon-suite.spec.ts` — full favicon contract (PR #78): asserts
  the Next 14 metadata API emits the exact `<link rel="icon|shortcut
icon|apple-touch-icon|manifest">` set with the expected hrefs, and
  that each referenced asset (`/favicon.ico`, `/favicon-16x16.png`,
  `/favicon-32x32.png`, `/apple-touch-icon.png`, `/site.webmanifest`)
  serves a 200 with the right MIME type.
- `issue-fixes.spec.ts` — regression tests captured from closed
  bugs; one spec per fix.

### Theme-generator runtime audits

The runtime contract that the lib's source-contract tests cannot
verify. These tests:

1. Navigate to `/theme-generator`.
2. Mutate a token via the editor UI (or directly on `:root` via
   `page.evaluate` for raw audits).
3. Read `getComputedStyle(consumer).<property>` on a live consumer
   element.
4. Assert the resolved value matches the override (within tolerance
   for color channels).

| Spec                                                | Coverage                                                                                                                                                                                                                                                                                                                           |
| --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `theme-generator-light-dark.spec.ts`                | mode toggle re-evaluates every consumer                                                                                                                                                                                                                                                                                            |
| `theme-generator-motion-propagation.spec.ts`        | `--duration-*` edits flow into `transition-duration` on Accordion / Progress consumers (regression guard for J-13a / AC-1)                                                                                                                                                                                                         |
| `theme-generator-sample-dialog.spec.ts`             | the in-page sample Dialog surface re-themes correctly                                                                                                                                                                                                                                                                              |
| `theme-generator-token-runtime-propagation.spec.ts` | high-value semantic + identity tokens propagate end-to-end                                                                                                                                                                                                                                                                         |
| `theme-generator-comprehensive-token-audit.spec.ts` | **64 parametric tests** across the 50 component-tracked tokens, generated from `_bmad-output/test-artifacts/component-token-map.md` (PR #62)                                                                                                                                                                                       |
| `tinted-memphis-shadow-runtime.spec.ts`             | regression guard for #66 — overrides intent tokens (`--primary`, `--success`, `--warning`, `--destructive`, `--info`) at runtime with `!important` and asserts each per-color `@utility shadow-memphis-{intent}` paints with the new tint (Button ghost, Input focus, Toast variants, Banner variants, Dialog danger, FeatureCard) |

The comprehensive audit is the canonical reference for "every
editable token must affect at least one component" and surfaced the
radius pill/full emit bug (PR #67) plus the rounded-sm /
rounded-selection / shadow-memphis-card preview-scene gaps tracked
as #64.

## Cross-browser stabilization patterns

Mid-2026-05 the suite hit several flakes specific to webkit. The
fixes converged on a small set of helpers used across runtime specs:

1. **`setTokenAndSettle(page, token, value)`** — sets a `:root`
   custom property via `page.evaluate`, awaits a one-frame
   `requestAnimationFrame` to let the cascade propagate, and waits
   on the consumer's transition-end event before sampling. Mirrors
   the manual interaction timing the editor would otherwise have.
2. **`parseLastNonTransparentRgb(boxShadow)`** — `box-shadow` strings
   contain multiple comma-separated layers and the order of
   `transparent` layers differs across browsers mid-transition. The
   helper picks the last opaque layer so assertions don't latch
   onto a transient frame.
3. **`channelsClose(actual, expected, tolerance)`** — RGB channel
   comparison with a small tolerance (`±2` per channel) absorbs
   sub-pixel rendering differences without hiding genuine drift.
4. **Decimal duration formatting (PR #57)** — webkit emits
   `transition-duration` like `0.00001s` (six decimals) where
   chromium emits `1e-05s`. The numeric helper
   `parseDurationToSeconds` parses both rather than string-matching.
5. **Important-flag override** — inline-element-style without
   `'important'` lost the cascade to `:root[data-palette='X']` in
   chromium. Runtime audits now set the property with `'important'`
   when simulating editor input, plus a `wait: 250ms` for Button's
   transition mid-flight.
6. **`<style>`-tag injection over inline `setProperty` (PR #66 /
   c4aa1ba)** — for tests that override CSS custom properties used
   inside `@utility` blocks, webkit refuses to honor a later
   `setProperty(...,'important')` if a stylesheet already declared
   the same `!important` value. The fix is to inject a `<style>` tag
   into `document.head` carrying the override declaration, which
   wins via author-stylesheet specificity rather than fighting the
   inline-vs-stylesheet ordering. Used by
   `tinted-memphis-shadow-runtime.spec.ts`.

These helpers live in the spec files (or a shared
`tests/scenarios/helpers/`) — keep them centralized to avoid drift.

## Running locally

```bash
# All projects (chromium + webkit)
pnpm --filter @damo/e2e test

# One browser
pnpm --filter @damo/e2e test --project=chromium

# One spec, headed, paused on first action (debugging)
pnpm --filter @damo/e2e test theme-generator-motion-propagation.spec.ts \
  --project=chromium --headed --debug

# Open the HTML report after a run
pnpm --filter @damo/e2e exec playwright show-report
```

The suite reuses an already-running `pnpm dev` outside CI; if no dev
server is up, Playwright boots one (120s timeout).

## Running in CI

CI workflow (`.github/workflows/ci.yml` — see also
`30-cross-cutting/10-ci-cd.md`):

- Runs both `chromium` and `webkit` projects sequentially after
  `lint-and-test` (the e2e job declares `needs: lint-and-test`).
- `retries: 2` swallows transient flakes without masking real
  regressions (failures still fail the run).
- Artifacts uploaded on failure: traces, videos, screenshots — plus
  the HTML report, retention 7 days.

**Performance optimizations (PR #90):**

- **Playwright browser cache** —
  `~/.cache/ms-playwright` is restored via `actions/cache@v4` keyed
  on `playwright-${{ runner.os }}-${{ hashFiles('pnpm-lock.yaml') }}`.
  On cache hit, the `Install Playwright browsers` step is skipped
  (~20s saved on GitHub-hosted runners). System deps
  (`playwright install-deps chromium webkit`) still run every time —
  apt-get state is not portable across runners.
- **Lib build artifact reuse** — `packages/ui/dist` is uploaded as
  the `damo-ui-dist` artifact from `lint-and-test` and downloaded
  in `e2e`, eliminating a duplicate `pnpm build` (~30–60s saved).
- **Production webserver** — Playwright boots `next start` against
  a fresh `pnpm --filter @damo/web build` instead of `next dev`
  (see Configuration above). Faster first request, no HMR overhead.
- **Concurrency cancellation** —
  `concurrency: { group: ci-${{ github.workflow }}-${{ github.head_ref || github.ref }}, cancel-in-progress: true }`
  cancels superseded PR runs. The `head_ref || ref` fallback is
  load-bearing: without `head_ref`, push and pull_request events
  for the same SHA resolve to different group keys and both runs
  proceed (defeating the optimization).

## Notes & gotchas

1. **Don't kill the dev server mid-run.** Several false failures in
   2026-05 traced back to `pnpm dev` being terminated while specs
   were in flight. The `reuseExistingServer` flag means a manually
   killed server simply leaves Playwright targeting a dead port.
2. **Webkit and chromium disagree on numeric formatting.** Anywhere
   the spec compares duration strings or shadow channel values,
   parse to numbers first.
3. **The comprehensive audit is auto-generated from a token map.**
   Editing the spec file directly without updating
   `_bmad-output/test-artifacts/component-token-map.md` will drift
   the test set away from the source of truth.
4. **No visual snapshot tests today.** Visual regressions are
   caught by humans, not by Playwright `toMatchSnapshot`. Adding
   visual snapshots is a candidate for a future iteration.
5. **Webkit-only flakes are quarantined, not patched-in-place
   (b11ce4c, 4bd889f, c4aa1ba, mid-2026-05).** Several specs
   surfaced webkit-specific cascade or shadow-painting glitches
   that don't reproduce on chromium. Rather than gate the whole
   suite on inconsistent behavior, the affected assertions are
   marked `test.skip(({ browserName }) => browserName === 'webkit',
'...')` with a comment linking the issue. Periodically revisit
   the quarantine list — webkit catches up on these eventually.

## Open questions

1. Add Firefox to the project list? Real-world traffic is
   non-trivial and webkit alone doesn't cover all gecko quirks.
2. Visual snapshot tier (Playwright `toMatchSnapshot` or Percy /
   Chromatic) — would catch the kind of "everything is silently
   wrong" bug that #58 and #66 represent.
3. The `helpers/` extraction has not happened yet — currently each
   spec carries its own copy of `setTokenAndSettle` etc. Centralize.
4. CI flakes still occasionally pass on retry — track flake rate
   and quarantine specs that exceed a threshold (e.g. 5%).
